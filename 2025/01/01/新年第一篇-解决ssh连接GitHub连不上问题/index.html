新年第一篇-解决ssh连接GitHub连不上问题

404joker404 Lv3

经过

本来说好不容易放假了，得把之前漏的都给填上，结果写了篇后，一hexo deploy却报错，一开始怀疑是不是代理出了问题还是不小心改过哪些文件，后来发现ssh -T git@github.com 报错超时，显示“ssh: connect to host github.com port 22: Connection timed out”

当时就纳闷了，之前不是早就弄好过了吗，于是去GitHub看了看密钥也没有问题

上面的哈希是私钥的哈希值，当时我误以为是公钥的哈希还浪费时间重新传了个

接下来我怀疑是不是22端口的问题，于是选择win+R wf.msc打开防火墙，发现也没人动它，又netstat -ano查看端口，也没有发现22端口的占用,就很奇怪了，也没法，于是开始了JSFW和问chatgpt

解决

chatgpt提供了一堆，不过大部分我都已经尝试过了，但JSFW有了结果

先在powershell里试了

ssh -T -p 443 git@ssh.github.com

没报timeout，成功收到“Hi QQdsb! You’ve successfully authenticated, but GitHub does not provide shell access.”

看来真可能是ISP把22端口给墙了，这电信真是个思慕东西的马匹

~/.ssh/config（没有的话就自己创建，我就是自建的）的内容如下

    Host github.com
    Hostname ssh.github.com
    Port 443
    User git

接下来再hexo deploy一下，成功了！
